> AI 학습 세션 워크플로 전체(모드·시작/중간/마무리·verifier)는 [`../agents/study/AGENTS.md`](../agents/study/AGENTS.md) 5장 참고.

## 노트 파일

- 파일명: **kebab-case 영어** (`redis-cache-strategy.md`).
- 카테고리 폴더: [`README.md`](README.md) 분류표에서 가장 맞는 폴더에 넣는다. 마땅한 게 없으면 새 폴더를 만들거나 `etc/`에 두고, **새 폴더를 만들면 README 분류표에 한 줄 추가**한다.
- 노트 상단: `# 제목` → **한 줄 정의/요약**(`> ` 인용) → 관련 노트가 있으면 `> 관련 문서: [제목](상대경로)` 링크. **상대경로 주의** — 다른 폴더면 `../폴더/파일.md`.
- 본문: 번호 매긴 섹션(`## 1.`, `## 2. …`) + 표·코드블록·다이어그램 적극 활용.
- 끝에: **📌 핵심 요약 / 🔗 참고 자료 / 🌱 심화 키워드 / ❓ 남은 질문** (독서·인문학이면 **📚 더 읽을거리 / 🔀 연결 주제**).
- 한 노트가 두 분야에 걸치면 **주 분야에 두고 다른 쪽에서 링크**로 연결.
- 백엔드 실무·면접에서 꺼내 쓰게 **"왜/언제/트레이드오프"** 중심으로.

## 학습 로그 — [`LOG.md`](LOG.md)

- **"커밋"**하면 그날 공부한 주제를 `LOG.md`에 기록한다(README에는 넣지 않는다).
- 구조: `## 연도` → `### 연-월` → `- **월-일 · 주제** — 한 줄 요약. [노트](상대경로)`.
  ```
  ## 2026
  ### 2026-07
  - **07-21 · JPA vs MyBatis** — 한 줄 요약. [노트](jpa/jpa-vs-mybatis.md)
  ```
- **새 달 첫 항목이면 `### 연-월`을, 새 해면 `## 연도`를 먼저 연다.** 이미 있으면 그 아래 항목만 추가.
- **항목은 한 줄** — 며칠(`MM-DD`) + 주제 + 한 줄 요약 + (있으면) 노트 링크. 상세는 노트에, 로그는 스캔용.
- 최신이 위로: **연·월·일 모두 역순(최신 우선)**.

## 저장 직전 검증 (필수)

- 노트를 커밋하기 전 [`../agents/study/verifier.md`](../agents/study/verifier.md) 체크리스트로 사실오류·오개념·뭉뚱그린 용어·확신도 미표기·깨진 링크를 잡는다. ❌·🔍가 남으면 고치거나 "확인 안 됨"으로 표기 후 저장.

## 트리거 요약

- **"커밋"** — LOG.md에 로그 기록 + 로컬 `git commit`까지 (push 안 함). 커밋 메시지 `YYYY-MM-DD 주제`.
- **"정리"** — verifier 검증 → 노트 저장 → 로그 기록 + commit → `git push origin master`까지. (기본 브랜치 `master`, 원격 `origin = devhong96/study`.)
- 커밋 메시지에 `Co-Authored-By:` 등 트레일러를 붙이지 않는다(이 레포 관례).
